# Documentation Hub

This `docs/` folder is the home for simulator documentation that supports both product usage and IaC authoring.

## Structure

| Folder | Purpose |
|---|---|
| `docs/components/` | One markdown file per simulator component, including properties, defaults, ranges, modes, and modeling notes. |
| `docs/iac/` | IaC-focused documentation, tutorials, examples, and reference material for the YAML DSL. |

## Suggested reading order

1. Start with `docs/iac/authoring-guide.md` for the big-picture workflow.
2. Use `docs/components/README.md` to jump to individual component docs.
3. Open the matching component page whenever you need default values, enum options, or YAML snippets.
